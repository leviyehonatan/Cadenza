## 1. Custom FL-style piano roll (`625a6ff`)

Replaced the vendored Sjhunt93 piano-roll (no usable keyboard, confusing
control panel) with a self-contained editor written for Cadenza:
`Source/UI/StylePartPianoRoll.{h,cpp}`.

- Piano keyboard down the left edge (white/black keys, octave labels); click a
  key to audition the pitch.
- **Click** empty grid to draw a note (drag to set length) · **drag body** to
  move · **drag right edge** to resize · **right-click** to delete · **wheel**
  to scroll the pitch range.
- Selectable grid snap (1/4…1/32 or Off, default 1/16), bar/beat ruler, orange
  playback marker. Opens on a useful register (GM percussion for drums, the
  playing range for melodic parts).
- Every edit writes back into the style so it's heard on the next loop pass.
- The vendored `Source/UI/PianoRoll/` folder (~18 files) and its CMake entries
  were deleted.

## 2. Recorder UX fixes (`e55b2dd`)

Three real bugs found from the user's screenshots:

- **Silent recording/playback** — an empty recorder section sent no channel
  setup, so the target part's MIDI channel had no voice (audition and the
  recorded loop were silent). `recorderPrepareTargetChannel()` now loads a kit
  (drums) or the role-default GM voice (melodic) + an audible CC7 on the part
  channel when the session starts, the part changes, and on arm.
- **Metronome had no control** — added a "Click" toggle to the recorder row
  (default on) and tightened the click to a short percussive tick (squared
  decay, ~14 ms) instead of a pitched beep.
- **Piano roll opened on the wrong notes** — now scrolls to a useful register
  by part type.

## 3. Loop-recording merge / dedup / quantize (`cd0d7ee`)

Recording and piano-roll edits share one quantize + merge path in
`StyleRecorder`:

- `quantizeTick` snaps to the grid (default 1/16) and folds into the loop.
- `mergeNoteIntoPart`: a drum hit on the same pitch + grid cell **updates** the
  existing note (newest velocity wins) instead of duplicating; a melodic note
  that overlaps an earlier same-pitch note **trims** that note (or drops it when
  fully covered).
- `commitTake` sorts, quantizes, and merges each note so overdub passes
  accumulate cleanly; `replacePartNotes` uses the same path so drawing obeys the
  same rules.
- New tests: loop wraparound, kick recorded twice (no duplicate), kick + hat
  (two notes), melodic overlap cleaned, quantize snaps recorded ticks.

## 5. Style Recorder (`593b8d5`)

The big one: **record your own styles, no Yamaha `.sty` needed.** New "Style
Recorder" row on the panel.

- **New** starts a looping empty section (1/2/4/8 bars) at the current tempo,
  metronome on; the whole keyboard plays the selected part (split off).
- **Part picker** — Drums, Bass, Chord 1/2, Pad, Phrase 1/2 — each on its
  standard SFF channel with the playback policy the equivalent Yamaha part would
  have (bass answers slash chords; chords fit the chord quality; phrases follow
  the root).
- **Record** loops and captures looper-style (each pass overdubs); clicking
  Record again commits. Note starts quantize, roles bake against the C-major
  source convention (root/3rd/5th/7th/color; drums absolute) so a recorded
  style follows live chords exactly like an imported one.
- **Clear Part** / **Save…** (writes a `.cstyle`) / **Exit**.
- Engine: `StyleRecorder` (pure C++, cadenza_core, full test suite) + a MIDI
  "capture mode" in `MidiRouter` that bypasses split/chord/melody routing while
  recording.

## 6. Drum / RTR fixes (`f617957`)

- RHY2 sub-rhythm (synth channel 8) now resolves to a real drum kit instead of
  a melodic voice ("English Horn"). SynthEngine forces the drum bank for both
  drum synth channels (8 + 9).
- RTR "…ToRoot" retrigger rules keep the bass register instead of jumping.

## 7. Full Yamaha chord vocabulary + NTT voicing + RTR rules (`e1f4cfb`)

One shared table (`Source/Midi/ChordTypes`) defines all 34 Yamaha chord types
and drives recognition and voicing end to end:

- Live recognition by Yamaha fingering rules (optional 5ths, subset matching):
  9ths/13ths, 6ths, add9, m7b5, altered dominants now detect. Ambiguities
  resolve like a real arranger (C-E-G-A = C6 with C in bass, Am7 with A in
  bass).
- Per-type voicing: chord-role notes take the type's own intervals; color/phrase
  tones snap to the type's chord scale (7(b9) → b9, maj7#11 → Lydian, …).
- FingeredOnBass slash chords drive bass parts to the named bass note.
- Sustained notes follow each part's Yamaha RTR rule across chord changes.
- Verified on the user's 891-file library (Genos2 + Balkan + 2 Dance packs):
  100% playable, 880 fully clean, 0 parse failures.
